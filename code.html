A Map of Computer Science

The Computer Science Ecosystem

A multilevel digital map of computer science that follows the assignment requirement: each area includes main activities, relations to other areas, important and open problems, important people, important venues, and both the local UVT dimension and the global dimension.

Team: Eduard Indrei · Eduard Belinschi · Anastasia Pleșa · Paul Dăcila · Christian Ciolac

Concept of the project

This project does not treat computer science as a flat list of topics. Instead, it presents the discipline as an ecosystem: areas have their own identities, but they continuously depend on each other. Algorithms support AI, software engineering turns ideas into code, systems run code at scale, databases organize information, cybersecurity protects trust, HCI connects systems to people, and computational science applies all these layers in real-world domains.

2. The contents of each area

Theory

Theory gives definitions, models, proofs, limits and abstractions. It explains what is possible, what is impossible and what resources are required.

  • Examples: complexity classes, semantics, relational algebra, learning theory, cryptographic models.
  • Theory explains why some problems are easy, some are hard and some require approximations.

Experiment

Experiment checks how ideas behave in practice through benchmarks, measurements, simulations, user studies and comparisons.

  • Code is tested for correctness, speed, memory use, robustness, security or usability.
  • Experiments connect areas because the output of one area often becomes the input of another.

Design

Design builds real artefacts: algorithms, languages, compilers, databases, interfaces, cloud services, AI systems and scientific software.

  • Design transforms theoretical ideas and experimental results into usable computing systems.
  • Good design also includes maintainability, safety, ethics and user-centred thinking.

What code or artefacts were tested?

The map includes concrete examples of experiments, not only abstract words. In practice, the experiments across the areas can involve code, datasets, simulations, network setups, interface prototypes or security labs.

Algorithmic and software code

C++ or Python implementations of sorting, graph routines, test suites, static-analysis tools, compilers or maintainability metrics.

Systems and networking setups

Client-server programs, network experiments, distributed services, cloud containers, throughput and latency measurements, fault-injection scenarios.

Data and AI pipelines

SQL queries, indexing strategies, search pipelines, machine-learning notebooks, training/evaluation scripts, explainability and bias tests.

Security, HCI and scientific workflows

Penetration-testing labs, encryption experiments, interface prototypes, usability studies, visual analytics, simulations compared with physical or domain data.

3. Area cards

01

Theory & Algorithms

the rules behind computation

Focus: Studies what can be computed, how efficiently it can be computed, and which abstract models and data structures make solutions possible.

Main activities

  • Theory: complexity classes, correctness proofs, automata, logic, graph theory and randomized methods.
  • Experiment: benchmark algorithm implementations, compare exact methods with heuristics, and study average-case behaviour on different inputs.
  • Design: create efficient algorithms and data structures for real-world constraints such as memory, time and energy.

Detailed experiments

  • Sorting and searching code can be tested on random, sorted and nearly sorted data to compare time complexity in practice.
  • Graph algorithms can be tested on sparse versus dense graphs to observe how theory meets real execution cost.
  • Approximation and heuristic methods can be evaluated when exact solutions are too slow.

Relations / connections

  • Feeds every other area: AI uses optimization, databases use indexing, systems use scheduling and security uses cryptography.
  • Has strong ties with mathematics, logic, combinatorics and operations research.
  • Provides the intellectual foundation for experiments carried out in software, systems and data-intensive areas.

Important / open problems

  • P versus NP: can every quickly verified solution also be quickly found?
  • Scalable algorithms for huge graphs, streaming data and online decision-making.
  • Approximation and randomization: how close to optimal is good enough when exact answers are expensive?
AT

Why this person matters

Alan Turing matters here because his 1936 model of the Turing machine gave computer science a precise way to define an algorithm. That model is the historical starting point for computability theory, decidability and the idea that some problems have formal limits.

Important venues

STOCFOCSSODAJACM

Global dimension

Globally, this area shapes both academic research and industrial computing: efficient algorithms power search engines, finance, logistics, bioinformatics and large-scale optimization worldwide.

Local UVT dimension

At UVT, this area connects to Theory of Computing and the theoretical computer science direction, including algorithms, logic, formal languages and automated reasoning.

02

Programming Languages & Software Engineering

turning ideas into reliable programs

Focus: Explores how software is specified, written, tested, verified, maintained and evolved over time.

Main activities

  • Theory: semantics, type systems, program logic, verification and abstraction mechanisms.
  • Experiment: compare debugging strategies, defect prediction models, testing coverage and maintainability metrics.
  • Design: compilers, APIs, frameworks, modular architectures, testing pipelines and developer tools.

Detailed experiments

  • Code can be tested with unit tests, integration tests and static analysis to measure robustness.
  • Different language features or frameworks can be compared through readability, performance and defect rates.
  • Version-controlled projects can be analysed to study technical debt, refactoring impact and bug evolution.

Relations / connections

  • Depends on theory for verification and on systems for runtime performance and deployment.
  • Influences cybersecurity because many vulnerabilities are caused by software bugs or unsafe APIs.
  • Connects directly to HCI because software quality is visible in usability, responsiveness and user trust.

Important / open problems

  • How do we build large systems that remain understandable after years of change?
  • Can practical tools prove important correctness properties without blocking development speed?
  • How can teams reduce bugs, security flaws and technical debt in long-lived systems?
GH

Why this person matters

Grace Hopper is important for programming languages because she worked on early compilers and pushed the idea that programs could be written closer to human language instead of only machine instructions. Her work helped lead to COBOL and to the modern idea of language tools that translate code for the computer.

Important venues

PLDIPOPLICSEFSE

Global dimension

Globally, this area supports the software industry, open-source communities, mobile and web ecosystems, and safety-critical applications from cars to medical devices.

Local UVT dimension

At UVT, students work heavily with programming and software engineering, while the Timișoara IT ecosystem creates a strong bridge between academic training and industrial projects.

03

Computer Systems, Networks & Cloud

the machines and infrastructure

Focus: Studies hardware-software interaction, operating systems, distributed systems, networks, cloud platforms and performance at scale.

Main activities

  • Theory: concurrency models, scheduling, distributed algorithms, fault tolerance and resource management.
  • Experiment: latency and throughput measurement, load tests, simulations, stress tests and fault-injection studies.
  • Design: operating systems, cloud services, network protocols, container platforms, IoT solutions and runtime infrastructures.

Detailed experiments

  • Network code can be tested by measuring latency, packet loss and throughput under different conditions.
  • Distributed systems can be evaluated by simulating crashes, delays and partitions to observe resilience.
  • Cloud or systems code can be benchmarked for scalability, energy consumption and resource usage.

Relations / connections

  • Provides the infrastructure for AI, databases, software deployment and computational science.
  • Relies on algorithms for scheduling, synchronization, routing and caching decisions.
  • Works closely with cybersecurity because system design directly affects trust and reliability.

Important / open problems

  • How can systems scale to millions of users while staying reliable and fast?
  • How do we reduce energy use in data centers, mobile devices and embedded platforms?
  • How can distributed systems remain correct when machines fail or messages are delayed?
DR

Why this person matters

Dennis Ritchie fits this area because he created the C programming language and co-developed UNIX with Ken Thompson. C became one of the main languages for operating systems and low-level software, while UNIX influenced how modern systems, servers and networks are organised.

Important venues

SOSPOSDISIGCOMMNSDI

Global dimension

Globally, this area keeps the internet, cloud computing, data centers, mobile infrastructure and large online services running at scale.

Local UVT dimension

At UVT, systems-related directions are visible through research interests such as cloud computing, high-performance computing and Internet of Things.

04

Data, Databases & Information Retrieval

organizing and finding knowledge

Focus: Deals with representing, storing, querying, protecting and interpreting large collections of structured and unstructured data.

Main activities

  • Theory: relational algebra, indexing, transactions, retrieval models, knowledge representation and ranking functions.
  • Experiment: query benchmarks, search relevance evaluation, data quality analysis and pipeline performance studies.
  • Design: database engines, warehouses, data pipelines, search systems, dashboards and analytics platforms.

Detailed experiments

  • SQL queries can be timed on different schemas or indexes to compare efficiency.
  • Search engines can be tested with precision, recall and ranking quality on benchmark collections.
  • Data cleaning and integration pipelines can be evaluated for accuracy, consistency and scalability.

Relations / connections

  • Feeds AI with training data and evaluation sets.
  • Depends on systems for storage and distributed execution, and on security for privacy and access control.
  • Connects to society through governance, transparency, digital archives and responsible data use.

Important / open problems

  • How do we keep data consistent across distributed systems?
  • How do we search, summarize and recommend information in massive, noisy and multilingual collections?
  • How can useful analytics respect privacy, fairness and legal constraints?
EF

Why this person matters

Edgar F. Codd is central to databases because he introduced the relational model: storing data in relations/tables and querying it through a formal mathematical foundation. This idea became the basis of relational database systems and strongly influenced SQL and modern data management.

Important venues

SIGMODVLDBICDESIGIR

Global dimension

Globally, this area is central to business intelligence, digital platforms, e-commerce, science, public administration and the management of large information ecosystems.

Local UVT dimension

At UVT, this area matches directions such as big data applications, data analysis and machine-learning applications on real datasets.

05

Artificial Intelligence, Machine Learning & Robotics

systems that learn and act

Focus: Builds systems that perceive, learn, reason, generate content, make predictions and sometimes act autonomously.

Main activities

  • Theory: learning theory, optimization, search, probabilistic reasoning, representation and planning.
  • Experiment: train/test comparisons, ablation studies, benchmark evaluation, bias analysis and explainability assessment.
  • Design: models, agents, recommendation systems, vision systems, NLP pipelines and robotic controllers.

Detailed experiments

  • Models can be trained on benchmark datasets and compared using accuracy, precision, recall or loss curves.
  • Ablation studies can test which features, model components or data choices really matter.
  • Explainability, robustness and fairness tests can show whether a model is reliable outside ideal conditions.

Relations / connections

  • Depends strongly on algorithms, data, optimization and computing infrastructure.
  • Raises security and privacy questions when models process sensitive or adversarial data.
  • Needs HCI when AI systems must explain decisions to human users.

Important / open problems

  • How do we make AI more trustworthy, robust and explainable?
  • How can models use less data and energy while keeping strong performance?
  • How do we reduce bias and avoid harmful or unsafe automated decisions?
GH

Why this person matters

Geoffrey Hinton is linked to AI because of his long work on neural networks, representation learning and deep learning. His research helped bring neural models back into the centre of machine learning and influenced the systems used today for vision, language and pattern recognition.

Important venues

NeurIPSICMLICLRAAAI

Global dimension

Globally, AI influences science, healthcare, finance, education, autonomous systems and creative industries, making it one of the most visible current areas of computer science.

Local UVT dimension

At UVT, the local AI dimension is especially visible through TRAIN and through research and teaching related to machine learning and explainable AI.

06

Cybersecurity & Privacy

protecting systems, data and people

Focus: Protects software, systems, communications and data against attacks while also defending privacy and trust.

Main activities

  • Theory: cryptography, threat models, formal security guarantees, authentication and privacy models.
  • Experiment: penetration testing, vulnerability assessment, attack simulation and measurement of defensive effectiveness.
  • Design: secure protocols, access-control systems, identity management, auditing and privacy-preserving mechanisms.

Detailed experiments

  • Security labs can test software and web applications for vulnerabilities under controlled conditions.
  • Encryption or authentication code can be benchmarked to compare security and performance trade-offs.
  • Privacy attacks and defences can be studied by measuring information leakage or re-identification risk.

Relations / connections

  • Cuts across software, systems, data and AI rather than staying isolated.
  • Depends on theory for cryptographic foundations and on HCI because users must understand security decisions.
  • Influences society directly through digital trust, regulation and safety.

Important / open problems

  • How do we secure increasingly complex and distributed systems?
  • How can privacy be preserved while still enabling useful data analysis or machine learning?
  • How do we make security usable enough that people actually follow it?
WD

Why this person matters

Whitfield Diffie belongs here because he helped create public-key cryptography with Martin Hellman. Their key-exchange idea changed cybersecurity by making secure communication possible even when two users had not shared a secret key beforehand, which is one foundation of secure internet communication.

Important venues

IEEE S&PUSENIX SecurityCCSNDSS

Global dimension

Globally, cybersecurity is essential for governments, companies, citizens and research infrastructures because digital systems cannot function without trust.

Local UVT dimension

At UVT, cybersecurity is one of the visible local research and teaching topics and is increasingly important for both academic projects and industry collaboration.

07

Human-Computer Interaction, Graphics & Visualization

where computing meets human experience

Focus: Studies how people interact with computing systems and how interfaces, graphics and visualizations shape understanding and use.

Main activities

  • Theory: interaction models, cognition, perception, accessibility and visual communication principles.
  • Experiment: usability tests, A/B experiments, eye-tracking studies, interviews and accessibility evaluation.
  • Design: interfaces, visualizations, educational tools, AR/VR environments and digital experiences.

Detailed experiments

  • Interface prototypes can be tested by measuring task completion time, error rate and user satisfaction.
  • Visualizations can be compared to see which one helps users understand data more correctly.
  • Accessibility testing can reveal whether systems work for users with different abilities and contexts.

Relations / connections

  • Connects software engineering to real users and data science to human understanding.
  • Influences AI explanations, cybersecurity usability and public trust in technology.
  • Often transforms technical outputs into dashboards, interfaces or visual stories that people can use.

Important / open problems

  • How do we make complex systems simple without hiding important risks?
  • How can interfaces be accessible, inclusive and understandable for diverse users?
  • How can visualization avoid misleading users while still simplifying complex information?
DN

Why this person matters

Don Norman fits HCI because his work made user-centred design and user experience a major part of computing. Instead of judging technology only by technical correctness, his approach studies how people understand, use and sometimes struggle with interfaces.

Important venues

CHIUISTCSCWSIGGRAPH

Global dimension

Globally, HCI and visualization influence education, design, health technology, public information systems, games and every product that depends on user experience.

Local UVT dimension

At UVT, this area is visible in teaching, student projects and public-facing digital tools, where software is judged not only by correctness but also by usability.

08

Computational Science & Applied Computing

computing as a tool for other sciences

Focus: Uses computing to model, simulate and solve problems from science, engineering, medicine, environment and society.

Main activities

  • Theory: numerical methods, modelling, simulation, optimization and uncertainty analysis.
  • Experiment: compare simulations with real measurements, run reproducibility studies and test scalability on HPC platforms.
  • Design: scientific software, digital twins, bioinformatics tools, geospatial pipelines and decision-support systems.

Detailed experiments

  • Simulation code can be validated by comparing predicted results with observed physical or biological data.
  • Parallel programs can be benchmarked on larger datasets to study speed-up and scalability.
  • Sensitivity analysis can show how much the final result changes when parameters or assumptions change.

Relations / connections

  • Depends on algorithms, data, systems and visualization.
  • Connects computer science with biology, geography, chemistry, economics, medicine and engineering.
  • Benefits from AI when models are combined with data-driven prediction or classification.

Important / open problems

  • How can simulations be accurate enough for decisions under uncertainty?
  • How can scientific code remain reproducible, maintainable and transparent?
  • How can AI help science without replacing validation, explanation and domain expertise?
JV

Why this person matters

John von Neumann connects to computational science through early stored-program computer architecture and the use of computation for mathematical and scientific problems. His work helped shape the idea that computers are not only calculation machines, but tools for modelling, simulation and applied research.

Important venues

SCSIAM CSEIPDPSIEEE eScience

Globally, this area supports climate modelling, genomics, physics, materials science, engineering design and many interdisciplinary research projects.

Local UVT dimension

At UVT, computational and applied mathematics plus domain-oriented research such as Earth observation make this area a strong bridge between informatics and other sciences.

4. The experiment web across the areas

STEP 1

Algorithms start the chain

Efficient routines, data structures and theoretical guarantees define what is worth implementing and testing.

STEP 2

Software turns ideas into code

Those routines become maintainable programs, libraries or tools that can now be tested automatically.

STEP 3

Systems run the code at scale

The same code is then evaluated under real workload, resource limits, concurrency and network conditions.

STEP 4

Data records the behaviour

Execution logs, query results and datasets are stored, cleaned and analysed to understand performance and correctness.

STEP 5

AI learns from the data

Machine-learning experiments depend on curated data, benchmark pipelines and robust evaluation protocols.

STEP 6

Security protects the chain

Security experiments test whether the code, systems, data and models remain trustworthy under attack or misuse.

STEP 7

HCI validates human use

Interfaces, explanations and visualizations are tested with people to check comprehension, safety and usability.

STEP 8

Computational science applies everything

In interdisciplinary problems, the whole chain reappears: algorithms, software, systems, data, AI, security and visualization are combined in one applied workflow.

Why this matters

This section makes the relations concrete: experiments are not isolated. A database benchmark can depend on systems tuning; an AI evaluation depends on data quality; a cybersecurity audit depends on the software architecture; a usability study depends on the interface built on top of all the previous layers.

Examples of linked experiments

Example 1: an algorithm is implemented in code, deployed on a server, tested under load, logs are stored in a database, then analysed by an ML model, secured against attacks and finally visualized for human interpretation. Example 2: a scientific simulation is parallelized, run on distributed hardware, validated with domain data and presented through an interactive dashboard.

5. Local and global dimensions

Local dimension: Department of Computer Science @ UVT

The local side of the map appears through teaching, student projects, seminars and research directions connected to UVT. Relevant local themes include theory of computing, software engineering, cloud computing, high-performance computing, AI and machine learning, cybersecurity, big data applications, computational and applied mathematics, and application-oriented work such as Earth observation.

Global dimension: international research communities

The global side is represented by international conferences, journals, professional societies and influential researchers. The areas in this map connect to global communities such as ACM, IEEE Computer Society, AAAI, USENIX and SIAM. This is important because local study and research at UVT do not exist in isolation: they are part of a wider global computer science conversation.
